## week of may 2, 2026

### new features

- **owned browser as a right-side sidebar** — an agent-controlled browser now lives inside the screenpipe window. start it from the new browser cards in **settings → browser**, then watch pi navigate, snapshot, and read pages without taking over your real browser. width and collapsed state persist per chat.
- **`/navigate` and `/snapshot` browser API** — control the owned browser from your own pipes or scripts. see the [API docs](/for-developers).
- **one-off scheduled pipes** — schedule a pipe to run once at a specific time with `schedule: at <iso>` in the manifest. see the [CLI reference](/cli-reference).
- **`screenpipe install <manifest-url>`** — install a pipe bundle directly from a manifest URL, no app required. see the [CLI reference](/cli-reference).
- **"pause for X minutes" tray menu** — pause recording from the tray with a submenu of presets; recording auto-resumes when the timer ends. the tray tooltip shows live status and a notification fires on pause and resume.
- **one-click delete of last 15 / 30 / 60 minutes** — quickly scrub the most recent recordings from the privacy menu. mp4, wav, and cached frames are all removed.
- **`--retention-mode` flag** — keep search alive while reclaiming disk by running in `media-only` mode. media is now the default retention target everywhere. see the [CLI reference](/cli-reference).
- **`screenpipe mcp --listen-on-lan` + `--api-key`** — expose the MCP HTTP server to other devices on your network with token auth. see the [MCP server docs](/mcp-server).
- **clipboard-capture privacy toggle** — opt out of clipboard capture from the UI or with a CLI flag. [learn more](/privacy-filter).
- **Tinfoil E2EE proxy routes** — chat traffic can now go through Tinfoil's encrypted enclave proxy, matching the [PII privacy filter](/privacy-filter) trust model.
- **meeting notes timeline scrubber** — replaces the per-row drawer with a full timeline view. scrub through the meeting, see the entire transcript, rename speakers from a popover, and jump to any moment in the timeline.
- **"coming up" calendar section** — meeting notes now shows your next meetings as cards and auto-enriches the active one. focused mode collapses the sidebar and autoselects the new meeting.
- **replay-the-moment + summarize bundle** — meeting receipts let you re-watch the exact moment something was said, plus a one-shot summary action.
- **meeting notes promoted to top-level sidebar** — two-pane notepad layout with focused mode, granola-style single column, and visible fetch errors.
- **queue follow-up prompts while pi is streaming** — type ahead while pi is still answering; queued messages send when the current turn ends.
- **connections moved to the main sidebar** — connections are now a top-level destination instead of buried under settings. see [connections](/connections).
- **new connections** — [Hermes Agent](/connections), Bee wearable, [Supabase OAuth](/connections), and [Warp MCP](/connections). Jira moved to OAuth 2.0 (3LO).
- **browser extension keepalive + extension polish** — MV3 service worker now keepalives every 30 seconds, eligible tabs are picked deterministically, and `chrome-extension://` pages are skipped.
- **single recording-quality knob** — replaces the previous quality matrix with one slider. fixes unreadable text on ultrawide monitors.
- **streaming onboarding summary** — onboarding now streams a "here's what i picked up" prose summary with thumbnails as engines come online, with a friendly fallback when the stream is quiet.
- **Help menu cards** — added **Documentation** and **Video tutorials** entry points.
- **a11y on macOS** — text events now carry `app_name` and `window_title` for richer downstream querying.
- **periodic desktop-app reminder for CLI users** — friendly nudge if you're running the CLI without the desktop app. silence with `SCREENPIPE_NO_REMINDERS`.
- **rewrite notification file links to the in-app viewer** — clicking a file link in a screenpipe notification now opens it inside the app instead of the OS handler.

### updates

- **AI presets** — first-run defaults are now seeded based on whether you're a pro user, and the OpenAI model list is fetched live instead of hardcoded.
- **sidebar polish** — collapse and search controls move into the icon column when collapsed, queued chat messages are de-emphasised, the first thought block is collapsed by default, and the live "writing…" dot pulses on the streaming chat row.
- **chat storage** — assistant messages are now persisted during streaming (not only on stream-end), atomic saves use unique tmp filenames, and chat renames broadcast across windows so every sidebar stays in sync.
- **scheduler** — pipes still in setup mode (missing connections) are skipped instead of erroring, and pipes can be disabled even when their connections are missing.
- **timeline search** — `⌘G` cycles matches, a dot strip shows hit density, and a gentle pulse highlights the current match. per-word highlights are now tight to the matched word.
- **tray health** — transient `/health` flaps no longer flash "Error" in the tray menu.
- **logs** — verbose ONNX Runtime INFO messages are suppressed, and unauthenticated WebSocket rejects are debug-level instead of warn.

### bug fixes

- **auth-key encryption toggle** — fixed an issue where toggling encryption could leave the engine with a stale key.
- **shortcut reminder size with encrypted store** — reminder window now sizes correctly when the secrets store is encrypted.
- **shortcuts** — pressing escape or clicking outside now cancels an in-progress recording, with clearer button labels.
- **retention** — "delete last N minutes" now actually unlinks the underlying mp4/wav files and drops cached frames.
- **theme** — resolved dark/light mode inconsistency in the Tauri integration.
- **docs UI contrast** — fixed low-contrast active states in dark mode.
- **notification settings link** — clicking it now navigates correctly.
- **updater** — non-fatal updater errors no longer bubble up to the caller.
- **meeting notes** — broken frame images are gone, the scrubber no longer ends early, the recording stop button works again, and the sidebar can be expanded during a focused meeting. status now streams via push instead of polling.
- **MCP `list-meetings`** — supports relative dates ("yesterday", "last monday", etc.).
- **MCP `merge-speakers`** — corrected JSON field names so the tool accepts your input.
- **audio** — detects zero-fill device hijack on input streams and only trips the watchdog after the stream was healthy. transient SCK callback warnings are downgraded.
- **Zoom detection** — no more false "meeting" trigger when the app is idle, and recording no longer terminates early when Zoom emits an audio-output signal.
- **speaker diarization** — all speakers are now seeded so long meetings don't fragment into ghost speakers.
- **Windows overlay** — escape now closes the overlay timeline when launched from search; console flashes during background `cmd`/`where` spawns are gone.
- **chat** — fixed hang at "analyzing…", strands at "writing…", lost messages on session switch, and the bus regression that broke pipe-watch.
- **sidebar** — duplicate pipe rows (already shown in scheduled) hidden, dark-mode rows now full opacity, scheduled-pipe order is stable FIFO, and the running-pipes store no longer re-renders every 30 seconds.
- **onboarding** — pipes run immediately after install, the live feed no longer gets stuck on quiet engines, and prose always renders locally before the LLM upgrade arrives.
- **connections** — Google Calendar dot only shows when actually connected, and a pipe-related side effect was fixed.
- **mDNS** — long hostnames are safely truncated to prevent panics.
- **macOS pasteboard** — serialized access on a dedicated dispatch queue to fix occasional stalls.


### new features

- **new integrations** — connect [Microsoft 365 and Teams](/connections), Google Docs, Google Sheets, QuickBooks Online, Bitrix24, Loops, Resend, Supabase, and the Pocket AI voice recorder. configure in **settings → connections**.
- **PII privacy filter** — opt-in filter (powered by a Tinfoil enclave) that redacts personal info from `/search` results and chat messages before they leave your machine. toggle from the chat composer. [learn more](/privacy-filter).
- **parallel chats with background streaming** — start multiple chats and switch between them; responses keep streaming in the background. a new chat sidebar shows pinned chats, recents, and pipes scheduled to run.
- **focus-aware capture** — when enabled, screenpipe only records the focused monitor instead of all of them, halving capture cost on multi-display setups. toggle from settings.
- **per-machine pipe favorites** — star pipes you use most. starred pipes sort to the top of the pipe list and get their own filter chip.
- **LAN-access toggle for the API** — bind the local API to `0.0.0.0` so other devices on your network can reach it. API auth is force-enabled when LAN access is on. see the [API docs](/for-developers).
- **`screenpipe sync remote`** — back up your screenpipe data over SSH/SFTP to a server you control. see the [CLI reference](/cli-reference).
- **`screenpipe db {check, recover, cleanup, unlock}`** — new CLI commands to repair database corruption and reclaim disk space. see the [CLI reference](/cli-reference).
- **`screenpipe logout`** — sign out of your screenpipe cloud account from the CLI.
- **connected apps as @mentions in chat** — type `@` in the chat filter popover to scope a question to a specific connected integration.
- **AI quota warnings** — heads-up notice when you're close to your daily limit on weighted models, so you don't get cut off mid-task.
- **browser extension v0.2** — new popup with connection status, an options page, and token-based auth.

### updates

- **CoreAudio Process Tap is now the default for system audio** on macOS 14.4 and later — better quality and no virtual driver required.
- **API keys are now readable and regeneratable** in the UI. user-set custom keys have been removed in favor of a single managed key.
- **Cursor-style inline edit** — click any of your past chat messages to edit and re-run it.
- **redesigned sidebar** — collapse button next to the macOS traffic lights, search opens a focused overlay window with a real keyboard shortcut, and starred pipes outrank running pipes in sort order.
- **timeline calendar** — empty days are now disabled in the day picker and skipped when navigating with arrow keys.
- **chat models hot-swap** — switching a preset's model no longer restarts the chat subprocess.

### bug fixes

- **mic stays connected on sleep/wake** — fixed false-positive disconnects.
- **Microsoft 365 / Teams OAuth** now uses the correct Azure client ID; personal Microsoft accounts are rejected up-front (Teams scope isn't supported there).
- **database reliability** — multibyte string truncation no longer panics, and connection starvation under heavy writes has been resolved.
- **diarization timeouts** — added a `speaker_id` index that fixes 60s+ stalls on long meetings.
- **memories pipeline** silently stopped updating — now back online.
- **macOS memory leaks** — wrapped clipboard capture, monitor enumeration, and focus-tracker callbacks in autorelease pools.
- **search results** — keyword flat mode no longer drops entries that lack text positions.
- **OAuth flows** — auto-refresh of expired tokens in the generic proxy, query params now forwarded through the connection proxy, broader Google Sheets scope, and a fix for tokens being dropped when reconnecting an instance.
- **notifications** — clicking "Open" on a native macOS notification now reliably brings the screenpipe window forward.
- **onboarding** — MCP install uses the bundled bun, so it works without a system bun on `PATH`. WhatsApp connection now resolves bun the same way.
- **calendar permission** — recovers when the macOS Privacy pane reports denied-but-empty.
- **chat polish** — drag-select restored in user messages, "+ new chat" reuses an empty chat instead of spawning duplicates, "Try again" appears when a model returns an empty response, and free models are always allowed regardless of quota.
- **Windows** — restored ARM64 release builds, switched consumer release to SSL.com EV signing, and propagated system root CAs to bundled bun/node via `NODE_EXTRA_CA_CERTS`.
