## week of may 3, 2026

### new features

- **meeting notes** — promoted to a top-level sidebar item with a two-pane notepad, a "coming up" calendar section, auto-enriched active meeting, and a focused mode that collapses the sidebar. new bundle includes receipts, replay-the-moment, and one-click summarize. queries both native and Google Calendar. [learn more](/meeting-transcription).
- **owned-browser sidebar** — embed an agent-controllable browser as a right-side sidebar inside screenpipe. configure user-browser vs. owned-default from **settings → browsers**.
- **new connections** — connect [Hermes Agent](/connections), the Bee wearable, [Jira](/connections) via OAuth 2.0 (3LO), [Supabase](/connections), and [Warp's MCP](/connections). the Connections panel has been promoted from Settings to the main sidebar.
- **tinfoil E2EE proxy** — new end-to-end-encrypted gateway routes for AI traffic through a Tinfoil enclave. see the [privacy filter docs](/privacy-filter).
- **MCP server LAN + auth** — `screenpipe mcp --listen-on-lan --api-key <key>` exposes the MCP HTTP server to other devices on your network with token auth. see the [MCP server docs](/mcp-server).
- **retention controls** — new `--retention-mode` CLI flag with a media-only mode that reclaims disk while keeping search alive, plus a one-click "delete last 15 / 30 / 60 minutes of recordings" action. see the [CLI reference](/cli-reference).
- **tray pause submenu** — "pause for X minutes" with auto-resume and a live tooltip showing recording status.
- **queued follow-ups in chat** — type and send the next prompt while the current response is still streaming; it runs as soon as the first turn finishes.
- **streaming onboarding recap** — first-run summary now streams "here's what i picked up" prose with thumbnails so the live feed never feels stuck.
- **timeline search polish** — ⌘G to jump between matches, a dot-strip for hit density, and a gentle pulse on the active hit. tighter per-word highlights too.
- **single recording quality knob** — one slider that fixes unreadable text on ultrawide displays without flooding disk.
- **in-app file viewer** — notifications with file links now open the file inside screenpipe instead of bouncing to the OS.
- **scheduled pipes (one-off)** — schedule a pipe to run once with `schedule: at <iso>`. see the [pipes docs](/pipes).
- **`screenpipe install <manifest-url>`** — install a bundle of pipes from a single manifest URL. see the [CLI reference](/cli-reference).
- **CLI app reminder** — periodic, dismissible nudge for CLI users to install the desktop app. opt out with `SCREENPIPE_NO_REMINDERS=1`.
- **help cards** — new Documentation and Video tutorials shortcuts in the Help menu.
- **macOS accessibility events** — text events now carry `app_name` and `window_title` so pipes can filter by app context.

### updates

- **AI model catalog** — OpenAI presets now fetch the live model list instead of hardcoding gpt-4 only. gpt-5.5 and gpt-5.5-codex added to the chatgpt fallback list.
- **chat tone** — pi defaults to plain prose and only flips to technical formatting when the question signals it.
- **smarter preset defaults** — pro vs non-pro accounts get different sensible defaults the first time we know who you are.
- **connection list** — `screenpipe connection list` now sorts connected services first.
- **notifications** — markdown links in the notification panel are no longer double-encoded; "Open" reliably brings the window forward.
- **sidebar** — collapse and search controls move into the icon column when collapsed; running-pipes sidebar no longer re-renders every 30 s; pipe rows already shown in "scheduled" are hidden from the running list.
- **windows polish** — title text hidden on every webview that has a title bar (login, Google Calendar OAuth, etc.); console flashes from background `cmd`/`where` spawns suppressed; ARM64 release builds restored.

### bug fixes

- **chat reliability** — user messages persist on send, stuck streams self-heal, pipe-watch no longer leaks events when switching chats, "writing…" no longer strands after a pipe completes, and aborting a chat actually deletes it.
- **meeting recording** — stop button works again, push-driven status stream replaces polling, and Zoom no longer terminates early on output-signal flaps or false-positive detection while idle.
- **owned-browser** — survives tray-only mode, picks any available parent window, plumbs cross-origin eval through a top-level `WebviewWindow`, and uses correct rust-side screen-coord math.
- **audio** — zero-fill watchdog only trips after the stream was healthy, and only on input streams; transient SCK "callback never fired" downgraded to a warning.
- **OCR / search** — Apple per-word records preserved on the level-0 fast path so per-word highlights line up with the timeline.
- **theme** — dark/light mode now consistent across Tauri webviews.
- **tray health** — transient `/health` flaps no longer flip the tray to "Error".
- **database** — write-queue treats SQLite I/O and corruption errors as fatal so they surface instead of silently looping.
- **mDNS** — long hostnames are safely truncated to prevent a panic from the mdns-sd library.
- **MCP `list_meetings`** — accepts relative date phrases like "yesterday" and "last monday".
- **`merge_speakers`** — corrected JSON field names so the tool actually merges.
- **AI gateway** — model alias is applied to the request body (not just provider selection); `reasoning_content` is promoted to `content` when the assistant returns empty content.
- **pipes** — disabling a pipe works even when its connections are missing; immediate run after install on onboarding; stale-install guard plus a UI cancel button.
- **browser extension** — picks eligible tabs, skips `chrome-extension://` pages, and an MV3 keepalive every 30 s prevents the service worker from sleeping mid-capture.
- **Google Calendar / connections** — connection dot reflects real status; `GET /connections/:id` recognizes browsers; loading + retry shown instead of a blank panel.
- **shortcut reminder** — correct size when the encrypted store is enabled.
- **updater** — non-fatal updater errors no longer propagate to callers.
- **windows escape key** — overlay timeline Escape now works when opened from search; routed correctly when Home has focus.
- **macOS pasteboard** — NSPasteboard access serialized on a dedicated dispatch queue to fix accessibility hangs.
- **keychain** — encryption is opt-in with a cache, and non-Mac platforms are gated cleanly during onboarding.
- **chat sidebar** — "+ new chat" opens a real new thread without snapping selection back to an old chat; renames broadcast across windows so the sidebar stays in sync.

## week of april 26, 2026

### new features

- **new integrations** — connect [Microsoft 365 and Teams](/connections), Google Docs, Google Sheets, QuickBooks Online, Bitrix24, Loops, Resend, Supabase, and the Pocket AI voice recorder. configure in **settings → connections**.
- **PII privacy filter** — opt-in filter (powered by a Tinfoil enclave) that redacts personal info from `/search` results and chat messages before they leave your machine. toggle from the chat composer. [learn more](/privacy-filter).
- **parallel chats with background streaming** — start multiple chats and switch between them; responses keep streaming in the background. a new chat sidebar shows pinned chats, recents, and pipes scheduled to run.
- **focus-aware capture** — when enabled, screenpipe only records the focused monitor instead of all of them, halving capture cost on multi-display setups. toggle from settings.
- **per-machine pipe favorites** — star pipes you use most. starred pipes sort to the top of the pipe list and get their own filter chip.
- **LAN-access toggle for the API** — bind the local API to `0.0.0.0` so other devices on your network can reach it. API auth is force-enabled when LAN access is on. see the [API docs](/for-developers).
- **`screenpipe sync remote`** — back up your screenpipe data over SSH/SFTP to a server you control. see the [CLI reference](/cli-reference).
- **`screenpipe db {check, recover, cleanup, unlock}`** — new CLI commands to repair database corruption and reclaim disk space. see the [CLI reference](/cli-reference).
- **`screenpipe logout`** — sign out of your screenpipe cloud account from the CLI.
- **connected apps as @mentions in chat** — type `@` in the chat filter popover to scope a question to a specific connected integration.
- **AI quota warnings** — heads-up notice when you're close to your daily limit on weighted models, so you don't get cut off mid-task.
- **browser extension v0.2** — new popup with connection status, an options page, and token-based auth.

### updates

- **CoreAudio Process Tap is now the default for system audio** on macOS 14.4 and later — better quality and no virtual driver required.
- **API keys are now readable and regeneratable** in the UI. user-set custom keys have been removed in favor of a single managed key.
- **Cursor-style inline edit** — click any of your past chat messages to edit and re-run it.
- **redesigned sidebar** — collapse button next to the macOS traffic lights, search opens a focused overlay window with a real keyboard shortcut, and starred pipes outrank running pipes in sort order.
- **timeline calendar** — empty days are now disabled in the day picker and skipped when navigating with arrow keys.
- **chat models hot-swap** — switching a preset's model no longer restarts the chat subprocess.

### bug fixes

- **mic stays connected on sleep/wake** — fixed false-positive disconnects.
- **Microsoft 365 / Teams OAuth** now uses the correct Azure client ID; personal Microsoft accounts are rejected up-front (Teams scope isn't supported there).
- **database reliability** — multibyte string truncation no longer panics, and connection starvation under heavy writes has been resolved.
- **diarization timeouts** — added a `speaker_id` index that fixes 60s+ stalls on long meetings.
- **memories pipeline** silently stopped updating — now back online.
- **macOS memory leaks** — wrapped clipboard capture, monitor enumeration, and focus-tracker callbacks in autorelease pools.
- **search results** — keyword flat mode no longer drops entries that lack text positions.
- **OAuth flows** — auto-refresh of expired tokens in the generic proxy, query params now forwarded through the connection proxy, broader Google Sheets scope, and a fix for tokens being dropped when reconnecting an instance.
- **notifications** — clicking "Open" on a native macOS notification now reliably brings the screenpipe window forward.
- **onboarding** — MCP install uses the bundled bun, so it works without a system bun on `PATH`. WhatsApp connection now resolves bun the same way.
- **calendar permission** — recovers when the macOS Privacy pane reports denied-but-empty.
- **chat polish** — drag-select restored in user messages, "+ new chat" reuses an empty chat instead of spawning duplicates, "Try again" appears when a model returns an empty response, and free models are always allowed regardless of quota.
- **Windows** — restored ARM64 release builds, switched consumer release to SSL.com EV signing, and propagated system root CAs to bundled bun/node via `NODE_EXTRA_CA_CERTS`.
